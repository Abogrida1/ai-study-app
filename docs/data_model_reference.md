# 📋 Data Model Reference — نموذج البيانات المرجعي

> هذا الملف هو المرجع الرئيسي لفهم كيف البيانات بتتحرك في النظام.  
> عدّل عليه وأنت بتبني النظام.

---

## 1. مبدأ الـ Authentication

> ❌ **مفيش Self Sign-up** — الطلاب مش بيسجلوا نفسهم.  
> ✅ **الجامعة هي اللي بتعمل الحسابات** — عن طريق ملف Excel يتحمّل من الـ Admin Dashboard.

### الفلو:
```
الأدمن بيرفع 4 ملفات Excel منفصلين (بالترتيب):

1️⃣ ملف المواد (courses.xlsx)
   → بيعمل: جدول courses + levels
   
2️⃣ ملف الطلاب (students.xlsx)
   → بيعمل: Supabase Auth + users + enrollments + sections

3️⃣ ملف الدكاترة (doctors.xlsx)
   → بيعمل: Supabase Auth + users + assignments (scope='course')

4️⃣ ملف المعيدين (tas.xlsx)
   → بيعمل: Supabase Auth + users + assignments (scope='section')

⚠️ الترتيب مهم — المواد الأول، بعدين الباقي!
```

## 2. الأدوار الثلاثة

### 👨‍🎓 الطالب (Student)
- بيسجل دخول بكود جامعي + باسورد
- بيشوف المواد المسجل فيها
- بيحضر المحاضرات (نظري) + السكاشن/العملي (لو المادة ليها)
- بتتعرض عليه درجاته

### 👨‍🏫 الدكتور (Doctor)
- **مسؤول عن مادته بالكامل**
- مسؤول عن **كل الطلاب** اللي بياخدوا المادة
- مسؤول عن **المحاضرات** (الجزء النظري)
- مسؤول عن **الدرجات**
- لو المادة **مفيهاش عملي/سكشن** → الدكتور هو المسؤول الوحيد عن كل حاجة

### 👨‍💻 المعيد (TA)
- **بيشتغل مع الدكتور** في مواد معينة
- مسؤول عن **السكشن فقط (= العملي/الـ Lab)**
- **مش مسؤول عن المحاضرات** — الدكتور بس
- بيرفع محتوى **خاص بالعملي فقط** (مش المحاضرات)
- مسؤول عن طلاب **السكشن بتاعه** بس (مش كل طلاب المادة)
- ✅ **ينفع يكون معيد لأكتر من مادة** — لكذا سنة ولكذا سكشن عادي
- مثال: م. محمد ممكن يكون معيد CS101 سكشن 1,2 + PHY101 سكشن 3 (سنة 1 و 2)

## 3. المفهوم الأساسي: السكشن = العملي

```
⚡ قاعدة ذهبية:
   سكشن (Section) = عملي (Practical/Lab)
   
   ✅ مادة ليها سكشن → ليها عملي → ليها معيد
   ❌ مادة مفيهاش سكشن → مفيهاش عملي → مفيهاش معيد → الدكتور بس
```

### مادة ليها عملي (has_sections = true):
```
مادة CS101 — مقدمة في علوم الحاسب
│
├── 📖 المحاضرات (النظري) ← الدكتور مسؤول
│   └── د. أحمد سعيد → يشرح لكل الطلاب في المادة
│   └── يرفع: فيديوهات المحاضرات، PDFs، واجبات نظري
│   └── مسؤول عن: الدرجات
│
└── 🔬 العملي/السكاشن ← المعيد مسؤول
    ├── سكشن 1: م. محمد علي
    │   ├── يرفع: محتوى العملي فقط (labs, exercises)
    │   └── طلاب: [أحمد، سارة، خالد] — سكشن 1 بس
    │
    ├── سكشن 2: م. نورا حسين
    │   ├── يرفع: محتوى العملي فقط
    │   └── طلاب: [ياسمين، عمر] — سكشن 2 بس
    │
    └── سكشن 3: م. أيمن خالد
        ├── يرفع: محتوى العملي فقط
        └── طلاب: [فاطمة، حسام] — سكشن 3 بس
```

### مادة مفيهاش عملي (has_sections = false):
```
مادة MATH201 — تفاضل وتكامل
│
├── 📖 المحاضرات (النظري) ← الدكتور مسؤول عن كل حاجة
│   └── د. منى إبراهيم → يشرح لكل الطلاب
│   └── مسؤول عن: المحتوى + الدرجات + كل حاجة
│
└── 🔬 عملي ← ❌ مفيش
    └── مفيش معيد — الدكتور هو المسؤول الوحيد
    
    ⚠️ الطلاب لسه ليهم سكاشن (للتنظيم) بس مش بيفرق هنا
       لأن مفيش عملي ومفيش معيد
```

## 4. ملخص المسؤوليات

| المسؤولية | الدكتور | المعيد |
|-----------|---------|--------|
| المحاضرات (النظري) | ✅ يشرح ويرفع | ❌ مش مسؤول |
| العملي/السكشن | ❌ | ✅ يشرح ويرفع |
| كل طلاب المادة | ✅ يشوفهم كلهم | ❌ يشوف سكشنه بس |
| الدرجات | ✅ مسؤول | ❌ |
| رفع محتوى المحاضرات | ✅ | ❌ |
| رفع محتوى العملي | ❌ | ✅ |
| مادة بدون عملي | ✅ مسؤول عن كل حاجة | ❌ مفيش معيد أصلاً |

## 5. الهيكل الهرمي

```
المستوى (Level) ← "السنة الأولى"
  └── السكشن (Section) ← "سكشن 1"، "سكشن 2"
        └── الطلاب ← [أحمد، سارة، محمود]
```

> ⚠️ **مفيش Groups** — الطلاب بيتقسموا على سكاشن مباشرة داخل المستوى.
> السكشن موجود دايماً كتقسيم إداري، لكن بيفرق فعلياً بس في المواد اللي ليها عملي.

## 6. خريطة التحويل: Excel → Supabase

```
Excel Student Row
  │
  ├──→ Supabase Auth (email: {university_id}@scholar.uni, password)
  │
  ├──→ public.users (id, university_id, full_name, role='student', level_id, section_id)
  │
  └──→ public.enrollments (user_id, course_id)  ← لكل مادة
       └──→ section_id ← سكشن الطالب (للمواد اللي ليها عملي)

Excel Doctor Row
  │
  ├──→ Supabase Auth (email: {university_id}@scholar.uni, password)
  │
  ├──→ public.users (id, university_id, full_name, role='doctor')
  │
  └──→ public.assignments (user_id, course_id, scope='course', role='doctor')
       └──→ لكل مادة بيدرّسها

Excel TA Row
  │
  ├──→ Supabase Auth (email: {university_id}@scholar.uni, password)
  │
  ├──→ public.users (id, university_id, full_name, role='ta')
  │
  └──→ public.assignments (user_id, course_id, scope='section', scope_id={section_id}, role='ta')
       └──→ لكل سكشن مسؤول عنه في كل مادة
```

## 7. الـ Database Schema (المعدّل)

```sql
-- المستويات
public.levels (id, name, name_ar, "order")

-- السكاشن — تابعة للمستوى مباشرة
public.sections (id, level_id, name, name_ar)

-- المستخدمين
public.users (id, university_id, full_name, role, level_id, section_id, ...)

-- المواد
public.courses (id, code, name, name_ar, credit_hours, level_id, semester, has_sections)
  -- has_sections: هل المادة ليها عملي ولا لا

-- تسجيل الطلاب في المواد
public.enrollments (id, user_id, course_id, section_id)

-- تعيين الدكاترة والمعيدين
public.assignments (id, user_id, course_id, scope, scope_id, role)
  -- Doctor: scope='course', role='doctor'  ← مسؤول عن كل المادة
  -- TA:     scope='section', scope_id=X, role='ta'  ← مسؤول عن سكشن معين

-- المحاضرات (النظري — الدكتور بيرفعه)
public.lectures (id, course_id, title, video_url, pdf_url, type='lecture', created_by)

-- محتوى العملي (المعيد بيرفعه)
public.lectures (id, course_id, title, video_url, pdf_url, type='practical', section_id, created_by)
  -- section_id: السكشن اللي المحتوى خاص بيه
  -- type: 'lecture' أو 'practical' للتفرقة
```

## 9. قواعد النظام

### 🔴 قواعد أساسية:
1. **الكود الجامعي فريد** — لكل الأدوار (طالب/دكتور/معيد)
2. **مفيش self-registration** — كل الحسابات من الأدمن
3. **السكشن = العملي** — مفيهاش سكشن يعني مفيهاش عملي
4. **المعيد مش بيرفع محاضرات** — بيرفع محتوى عملي فقط
5. **الدكتور مش بيرفع عملي** — بس المحاضرات والدرجات
6. **المعيد يشوف طلاب سكشنه بس** — الدكتور يشوف كل الطلاب
7. **مفيش معيد لمادة بدون عملي** — الدكتور المسؤول الوحيد
8. **مفيش Groups** — التقسيم بالسكاشن مباشرة

### 🟡 أسئلة تم الإجابة عليها:
- [x] هل الطالب يقدر يغير الباسورد بعد أول login؟ ← **لا — الباسورد ثابت من الجامعة**
- [x] هل الدكتور يقدر يضيف معيدين من الأبليكيشن ولا بس من الأدمن؟ ← **لا — بس من الأدمن**
- [x] هل فيه admin accounts ثابتة ولا بتتعمل برضو من Excel؟ ← **الأدمن يقدر يعدل كل حاجة — حسابات ثابتة**
- [x] هل معيد ممكن يكون معيد لأكتر من مادة؟ ← **أيوا — لكذا مادة ولكذا سنة ولكذا سكشن**

### 🔴 قواعد مؤكدة إضافية:
9. **الطالب مش بيغير باسورده** — الباسورد من الجامعة وثابت
10. **الدكتور مش بيضيف معيدين** — كل التعيينات من الأدمن بس
11. **الأدمن يتحكم في كل حاجة** — حسابات، مواد، تعيينات، كل شيء

## 10. تجربة الطالب في الأبليكيشن (Student View)

### 📱 الشاشة الرئيسية — الطالب بيشوف:
- **مواده فقط** — المواد المسجل فيها بس (من enrollments)
- **سكشنه** — ظاهرله هو في سكشن كام
- **مفيش مواد طلاب تانيين** — كل طالب يشوف حاجته بس

### 📚 لما الطالب يفتح مادة معينة:

```
مادة CS101 — ماده ليها عملي (has_sections = true)
│
├── 📖 تاب المحاضرات (Lectures)
│   └── كل المحاضرات اللي نزلها الدكتور بتاع المادة دي
│   └── فيديوهات، PDFs، واجبات نظري
│   └── 🔓 متاحة لكل الطلاب المسجلين في المادة
│
└── 🔬 تاب العملي (Practical/Section)
    └── محتوى العملي اللي نزله المعيد بتاع سكشنه
    └── labs, exercises, assignments عملي
    └── 🔒 الطالب يشوف بس محتوى السكشن بتاعه
        (طالب في سكشن 1 يشوف محتوى سكشن 1 بس)
```

```
مادة MATH201 — ماده مفيهاش عملي (has_sections = false)
│
├── 📖 تاب المحاضرات (Lectures)
│   └── كل المحاضرات اللي نزلها الدكتور
│   └── 🔓 متاحة لكل الطلاب المسجلين
│
└── 🔬 تاب العملي ← ❌ مش موجود (أو مخفي)
    └── المادة مفيهاش عملي — مفيش حاجة هنا
```

### 💬 الشات — الطالب يقدر يتكلم مع:
| مين | إزاي |
|-----|-------|
| دكتور المادة | ✅ شات في كل مادة مسجل فيها |
| معيد السكشن | ✅ بس المعيد بتاع سكشنه في المادة |
| ❌ دكتور مادة تانية | ❌ مش مسجل فيها |
| ❌ معيد سكشن تاني | ❌ مش في سكشنه |

### 🔒 قواعد العرض للطالب:
1. **المواد**: يشوف بس المواد المسجل فيها `WHERE enrollments.user_id = me`
2. **المحاضرات**: يشوف محاضرات مواده بس `WHERE lectures.course_id IN (my_courses) AND type='lecture'`
3. **العملي**: يشوف عملي سكشنه بس `WHERE lectures.course_id IN (my_courses) AND type='practical' AND section_id = my_section`
4. **الشات**: يشوف الدكاترة والمعيدين بتوع مواده وسكشنه فقط
5. **الدرجات**: يشوف درجاته هو بس

### 📊 مثال عملي — الطالب أحمد (سكشن 1):

```
أحمد بيشوف:
├── CS101 (ليها عملي)
│   ├── 📖 محاضرات: كل اللي نزله د. أحمد سعيد
│   ├── 🔬 عملي: بس اللي نزله م. محمد علي (معيد سكشن 1)
│   │   └── ❌ مش بيشوف عملي م. نورا حسين (معيدة سكشن 2)
│   └── 💬 شات: يكلم د. أحمد + م. محمد بس
│
├── MATH201 (مفيهاش عملي)
│   ├── 📖 محاضرات: كل اللي نزلته د. منى إبراهيم
│   ├── 🔬 عملي: ❌ مفيش
│   └── 💬 شات: يكلم د. منى بس
│
└── PHY101 (ليها عملي)
    ├── 📖 محاضرات: كل اللي نزله د. سامي
    ├── 🔬 عملي: بس اللي نزله م. أيمن (معيد سكشن 1)
    └── 💬 شات: يكلم د. سامي + م. أيمن بس
```

## 11. تجربة الدكتور في الأبليكيشن (Doctor View)

### 📌 الدكتور ينفع يدرّس أكتر من مادة لأكتر من سنة

### 📱 فلو التنقل:

```
الدكتور يفتح الأبليكيشن
  │
  ├── 📋 صفحة اختيار السنة (Level)
  │   ├── سنة 1
  │   ├── سنة 2
  │   └── سنة 3
  │
  └── يختار سنة → 📋 صفحة اختيار المادة (Course)
      ├── CS101 — مقدمة في علوم الحاسب
      ├── CS201 — هياكل البيانات
      └── ...
      │
      └── يختار مادة → 📄 صفحة إدارة المادة
          ├── 📖 المحاضرات — يرفع/يعدل/يحذف
          ├── 📊 الدرجات — يتابع درجات كل الطلاب
          └── 💬 الشات — يتواصل مع الطلاب
```

### 📄 داخل صفحة المادة — الدكتور يقدر:

| الإمكانية | التفاصيل |
|-----------|----------|
| 📖 رفع محاضرات | فيديوهات (Google Drive)، PDFs، ملفات |
| 📖 تعديل/حذف محاضرات | أي محاضرة رفعها |
| 📊 متابعة الدرجات | يشوف كل طلاب المادة ودرجاتهم |
| 📊 إدخال درجات | عملي، شفوي، ميد تيرم، فاينال |
| 💬 شات مع الطلاب | كل الطلاب المسجلين في المادة |
| 👥 قائمة الطلاب | يشوف كل الطلاب المسجلين وسكاشنهم |

### مثال عملي — د. أحمد سعيد:

```
د. أحمد سعيد بيشوف:
│
├── 📂 سنة 1
│   └── CS101 — مقدمة في علوم الحاسب
│       ├── 📖 المحاضرات: يرفع ويعدل محاضرات النظري
│       ├── 📊 الدرجات: يدخل/يعدل درجات كل الطلاب
│       ├── 👥 الطلاب: [أحمد، سارة، خالد، ياسمين، عمر، فاطمة، حسام]
│       └── 💬 شات: يكلم أي طالب في المادة
│
└── 📂 سنة 2
    └── CS201 — هياكل البيانات
        ├── 📖 المحاضرات: يرفع ويعدل محاضرات النظري
        ├── 📊 الدرجات: يدخل/يعدل درجات كل الطلاب
        ├── 👥 الطلاب: [مروان، هدى، ...]
        └── 💬 شات: يكلم أي طالب في المادة
```

## 12. نظام الدرجات (Grading System)

### 📊 توزيع الدرجات (من 100):

| المكوّن | بالإنجليزي | الدرجة | مين بيدخله | ملاحظات |
|---------|------------|--------|-----------|---------|
| الميد تيرم | Midterm | **/15** | الدكتور | امتحان نص الترم |
| الشفوي | Oral | **/10** | الدكتور | امتحان شفوي |
| العملي | Practical | **/10** | الدكتور | درجة العملي/الـ Lab |
| حضور وتاسكات | Attendance | **/5** | الدكتور | حضور وغياب + واجبات |
| الفاينال | Final | **/60** | الدكتور | الامتحان النهائي |
| **المجموع** | **Total** | **/100** | — | — |

### هيكل الدرجات في الـ Database:

```sql
public.grades
  - id UUID
  - enrollment_id UUID → enrollments  (الطالب + المادة)
  - midterm NUMERIC(4,1)     ← /15  الميد تيرم
  - oral NUMERIC(4,1)        ← /10  الشفوي
  - practical NUMERIC(4,1)   ← /10  العملي
  - attendance NUMERIC(3,1)  ← /5   حضور وتاسكات
  - final NUMERIC(4,1)       ← /60  الفاينال
  - total NUMERIC(5,1)       ← /100 المجموع (يتحسب تلقائي)
  - grade_letter TEXT         ← التقدير (A, B, C, D, F)
  - updated_by UUID → users
  - updated_at TIMESTAMPTZ
```

### 📊 الدكتور بيشوف جدول الدرجات كده:

```
مادة CS101 — درجات الطلاب:
```
